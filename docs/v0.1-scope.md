# DragonBoat v0.1 Scope

DragonBoat v0.1 is a focused private-preview milestone.

It should prove that heterogeneous coding agents can coordinate through shared tasks, peer messages, isolated worktrees, and evidence bundles while the user remains in control.

## v0.1 Promise

In one local repository, DragonBoat should demonstrate a full-stack software task where:

- Codex acts as the steerer.
- Claude Code acts as multiple rower workers.
- Each rower receives a bounded task packet.
- Each rower works in an isolated worktree.
- Rower-to-rower handoffs flow through a peer mailbox.
- Work is recorded as local events.
- Each rower returns an evidence bundle.
- The steerer reviews the evidence before accepting the result.
- A local web command deck can show the process live and replay it later.

## Initial Demo Crew

- **Steerer**: Codex CLI, configured locally by the user.
- **Rower 1**: Claude Code CLI as frontend worker.
- **Rower 2**: Claude Code CLI as backend worker.
- **Rower 3**: Claude Code CLI as QA/Ops worker.

The user's local Claude Code setup may use a third-party or domestic model provider. DragonBoat should not care. The point is user-owned model routing.

## Initial Demo Task

The first demo should use a newly created, minimal full-stack web app.

The task should be realistic enough to require frontend, backend, and QA/Ops coordination, but small enough to finish in a controlled demo.

## Non-Goals

v0.1 should not promise:

- a generic agent framework
- full autonomous software development
- a cloud-hosted control plane
- a replacement for Codex or Claude Code configuration
- hard real-time delivery across every agent platform
- support for every coding agent on day one
- a full IDE or code editor
- automatic merge without steerer/user review
- shared-worktree concurrent editing

## Delivery Semantics

DragonBoat should support soft real-time coordination:

- use hooks or plugins where a platform supports them
- use polling where a platform does not
- keep all important state in a local event log
- make delivery state visible rather than pretending everything is instant

## Local Web Command Deck Scope

The v0.1 command deck is a local web app and replay surface.

It should show:

- crew roster
- task graph
- live status
- mailbox timeline
- evidence bundles
- agent console output
- event stream
- run replay

It should not configure the user's Codex or Claude Code accounts in v0.1.

A packaged desktop app can wrap this local web command deck later, but native desktop packaging is not required for the first private preview.

## Success Criteria

v0.1 succeeds if a viewer can understand, within a few minutes:

- who the steerer is
- who the rowers are
- what each rower was asked to do
- how rowers communicate
- what evidence was produced
- why the final result was accepted or rejected
