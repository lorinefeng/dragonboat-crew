# DragonBoat v0.1 Protocol And Adapters Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the smallest DragonBoat runtime that can record a local run, hand task packets from Codex to Claude Code rowers, collect evidence bundles, and feed a command-deck replay.

**Architecture:** DragonBoat core owns local files, schemas, event append, mailbox materialization, and validation. Codex and Claude Code are adapters that consume and produce DragonBoat contracts; they do not define the core model.

**Tech Stack:** TypeScript core, JSON Schema Draft 2020-12, append-only NDJSON events, Codex CLI adapter, Claude Code CLI adapter, Vite + React + TypeScript demo app, Hono Node backend, Vitest, Playwright, local web command deck. Tauri is deferred as a future packaging option.

---

## File Structure

- `schemas/v0/*.schema.json`: machine-readable data contracts.
- `fixtures/v0/`: valid and invalid sample task packets, events, and evidence bundles.
- `packages/core/src/`: run store, schema validation, task packet rendering, evidence rendering, mailbox primitives.
- `packages/adapters/codex-cli/src/`: steerer adapter for `codex exec`.
- `packages/adapters/claude-code-cli/src/`: rower adapter for `claude --print`.
- `apps/demo-web/`: minimal full-stack web app for the first crew run.
- `apps/demo-web/`: local web command deck, event stream, agent console, and first full-stack demo surface.
- `docs/`: product, protocol, adapter, and demo documentation.
